# Codex Working Rules

## Project

You are working on Core — the internal product, inventory and rental system for 2010shop.

Read `/docs` before making architectural changes.

## Golden rules

1. Core is the Single Source of Truth.
2. Do not change architecture silently.
3. Keep routes thin.
4. Put business logic in services.
5. Put database access in repositories.
6. Do not change stock directly; always use StockMovement.
7. Do not put prices inside Variant; use Price.
8. Do not put SKU, barcode or stock inside Product.
9. Do not create ProductImage / VariantImage tables; use Image + ImageLink.
10. Keep integrations isolated.

## Development rules

- Use Python 3.13.
- Use uv.
- Use FastAPI.
- Use SQLAlchemy 2.
- Use Alembic.
- Use PostgreSQL.
- Use SQLAdmin for MVP admin.
- Use Docker Compose for local run.
- Use Angie as the only reverse proxy.
- Do not add Nginx.
## Work style

- One task at a time.
- Small commits.
- Prefer simple code.
- Do not add dependencies without a reason.
- Do not build future features unless asked.
- MVP first.

## Before finishing a task

Run:

```bash
uv run ruff check .
uv run pytest
```
If commands fail, explain why.
