# Core Product

## Что такое Core

Core — операционная платформа для небольших магазинов и сервисов аренды.

Она управляет жизненным циклом физических объектов: от первой фотографии и регистрации до поступления, подготовки к продаже, продажи, повторной аренды, возврата, обслуживания и анализа работы команды.

Core не является бухгалтерией, универсальной ERP или обязательным пользовательским интерфейсом. Он предоставляет бизнес-процессы и документированный API, поверх которых могут работать SQLAdmin, мобильный интерфейс, собственный frontend, мессенджеры и внешние интеграции.

## Для кого создаётся продукт

Core ориентирован на владельцев небольших магазинов и сервисов аренды, которым нужно:

- быстро вводить новые объекты в учёт с телефона;
- передавать ежедневные операции сотрудникам;
- сохранять автора и историю каждого значимого действия;
- видеть узкие места и результаты работы команды;
- не зависеть от интерфейса или одной внешней платформы;
- разворачивать систему самостоятельно, использовать её по подписке или в составе франшизы.

## Продуктовая цель

Core должен позволить владельцу перестать быть незаменимым участником ежедневных операций.

Новый сотрудник должен быстро освоить понятные рабочие режимы, а владелец — видеть:

- что было сделано;
- кто это сделал;
- сколько времени занял процесс;
- где произошли ошибки, отмены или повторные действия;
- что требует внимания.

## Два равноправных жизненных цикла

### Продажа

```text
Фото → Каталог → Поступление → Подготовка к продаже → Продажа
```

### Аренда

```text
Фото → Каталог → RentalAsset → Выдача → Возврат → Осмотр → Обслуживание → Следующая выдача
```

Продажа и аренда используют общую основу: каталог, варианты, изображения, идентификаторы, пользователей, аудит и историю операций. Аренда является первоклассным жизненным циклом Core, а не внешним дополнением.

## Workflows, а не CRUD

Ежедневная работа происходит через workflows: приёмку, подготовку к продаже, продажу, выдачу и возврат аренды, инвентаризацию.

Карточки Product, Variant, RentalAsset и документов служат справочником, точкой просмотра состояния и истории. Они не должны превращаться в основной рабочий экран, который сотрудник постоянно открывает и редактирует вручную.

Core не требует отдельного универсального BPMN-движка. Workflow — это понятный пользовательский путь и координация существующих доменных сервисов.

## API как продуктовый контракт

Любая полноценная бизнес-возможность сначала должна быть доступна через документированный API.

Интерфейс — один из клиентов Core. Партнёр может создать собственный web, mobile, kiosk или messenger-интерфейс, не переписывая правила ядра.

## Модель поставки

Core должен поддерживать:

- самостоятельное развёртывание;
- управляемый доступ по подписке;
- поставку в составе франшизы;
- подключение собственного интерфейса через API.

Путь от подготовленного сервера до работающей системы должен быть понятным и воспроизводимым. Документация для пользователя, администратора, разработчика API, партнёра и инвестора является частью продукта.

## Продуктовый ориентир

> Core помогает не хранить данные, а управлять жизненным циклом физических объектов и работой магазина.
