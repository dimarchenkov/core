# Core Business Processes

## MVP scope

В первой версии Core описываются только процессы, которые нужны для запуска учета:

1. Приемка товара с телефона.
2. Импорт существующих товаров из Tilda CSV.
3. Подготовка и отправка товаров в AQSI.
4. Базовый процесс аренды оборудования.

Все остальные процессы считаются будущими расширениями.

---

# 1. Приемка товара с телефона

## Цель

Быстро создать товар в Core в момент поступления.

Главное правило: товар не должен попасть в учет без фото.

## Участник

- Manager

## Основной сценарий

1. Manager открывает Core с телефона.
2. Нажимает “Новая приемка”.
3. Делает фото товара или выбирает фото из библиотеки.
4. Вводит название товара.
5. Выбирает категорию.
6. Указывает количество.
7. Указывает источник поступления.
8. Указывает закупочную цену.
9. Указывает розничную цену.
10. Сохраняет карточку.

## Результат

Core создает:

- Product
- Variant
- Image
- Price
- StockMovement
