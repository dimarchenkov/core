# v0.4.0 — Ready for Sale

## Outcome

Core can now turn a received CatalogVariant into a sellable item with an attributed retail price, generated barcode, explicit readiness state, printable label and verified AQSI publication.

## Added

- immutable retail price history with current-price lookup;
- automatic internal EAN-13-compatible barcodes;
- exact Variant lookup by scanned barcode;
- derived Ready for Sale checks;
- printer-independent 58 × 40 mm PDF labels;
- asynchronous and idempotent AQSI publication;
- attributed publication attempt history;
- automatic discovery of a single AQSI shop;
- deterministic `Товары Core` AQSI category;
- AQSI product read-back verification;
- delayed background retries through the RQ scheduler.

## AQSI boundary

The first integration publishes ordinary piece goods with:

- current retail RUB price;
- primary barcode and SKU;
- VAT code `6` — VAT exempt;
- one configured or safely discovered shop.

AQSI stock, sales, returns, images and rental operations remain outside this release.

## Verified

- database migrations through `0014_create_publications`;
- Docker API, PostgreSQL, Redis and worker startup;
- 179 automated tests;
- one real product published and read back from AQSI;
- generated barcode recognized by the cash register;
- repeated publication command did not create a duplicate job.

## Smoke-test findings fixed

- standalone worker now registers identity models required by audit foreign keys;
- worker starts the RQ scheduler so delayed retries execute;
- AQSI's canonical `шт.` read value is treated as equivalent to the submitted `Штука` unit.

## Deferred

- versioned runtime AQSI VAT profiles and controlled bulk republishing;
- physical XPrinter XP-365B calibration and direct TSPL-over-USB printing;
- mobile photo-first workflow UX;
- sales and rental lifecycles.
