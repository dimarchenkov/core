# v0.5.0 — Workflow UX

Core now provides its first complete employee workflow: receive goods from a phone, preserve the
work as a resumable draft and atomically post the resulting Receipt to the inventory ledger.

## Added

- employee-owned resumable Intake sessions and item drafts;
- Photo First creation for new catalog positions;
- barcode, SKU and Variant identification for repeat deliveries;
- atomic Intake completion across Catalog, Media, Receipt and Inventory;
- derived missing requirements for drafts and Ready for Sale results;
- employee activity events and personal activity feed API;
- Ready for Sale attention queue API;
- authenticated source-image delivery;
- dependency-free first-party mobile interface at `/app`.

## Changed

- Supplier selection happens after physical-item identification;
- repeat deliveries reuse the catalog photo and do not require a new image;
- transaction ownership is explicit at command and workflow boundaries;
- the legacy one-shot Intake endpoint is deprecated;
- unsaved form values are persisted before workflow transitions;
- completion results explain exact Ready for Sale follow-up requirements.

## Database

- `0015_create_intake_sessions`;
- `0016_create_activity_events`.

Run `uv run alembic upgrade head` as part of every deployment before using the new application
version.

## Verified

- 218 automated tests;
- Docker rebuild and migration to `0016_create_activity_events`;
- controlled phone smoke test through Angie;
- real Receipt `REC-000007` posted with an immutable `+200` stock movement;
- source photo, generated SKU and barcode, actor attribution and completed IntakeSession;
- clean API logs after deployment.

## Known follow-up

- purchase price remains a Receipt fact and never becomes retail price automatically;
- a new Variant remains in Ready for Sale attention until a positive retail RUB price is assigned;
- further visual and interaction refinements remain normal backlog work;
- the first-party Ready for Sale queue screen is still pending.
