# Rental

## Назначение

Rental управляет конкретными физическими экземплярами, предназначенными для аренды.

Объектом аренды является `RentalAsset`, а не `Product` и не `Variant`.

```text
Product
└── Variant
    └── RentalAsset
```

- `Product` описывает семейство товара.
- `Variant` описывает конкретную товарную позицию и является единицей количественного складского учета.
- `RentalAsset` идентифицирует одну конкретную физическую вещь.

Например:

```text
Product: Шуруповерт
Variant: Makita DF333, комплект с кейсом
RentalAsset: RENT-000017
```

## RentalAsset

Один `RentalAsset` всегда представляет ровно одну физическую вещь.

У `RentalAsset` отсутствует `quantity`. Количество и складские движения принадлежат Inventory и
учитываются на уровне `Variant`.

`RentalAsset` хранит стабильную идентичность экземпляра и его текущее состояние. Текущие поля
нельзя присваивать произвольно: они изменяются только через доменные команды, проверяющие
инварианты.

### Создание при приемке

`RentalAsset` создается при завершении Intake для экземпляров арендуемого `Variant`.

Эту межконтекстную операцию выполняет `CompleteIntakeWorkflow`:

1. завершает приемку;
2. проводит складские изменения через Inventory;
3. создает необходимое количество отдельных `RentalAsset`;
4. фиксирует результат одной общей транзакцией.

Rental и Inventory не координируют друг друга напрямую. Их доменные сервисы участвуют в операции
без `commit()` и `rollback()`; владельцем транзакции является workflow.

## Состояние

Текущее состояние экземпляра разделено на три независимые оси.

### AssetPurpose

- `RENTAL` — экземпляр находится в прокатном фонде;
- `SALE` — экземпляр выведен из проката и подготовлен к будущей продаже;
- `RETIRED` — экземпляр окончательно выведен из оборота.

`RETIRED` является терминальным состоянием. Обычные доменные команды не могут вернуть такой
экземпляр в `RENTAL` или `SALE`.

### AssetCondition

- `NEW` — новый экземпляр;
- `GOOD` — исправен, нормальные следы эксплуатации;
- `FAIR` — исправен, но заметно изношен;
- `DAMAGED` — поврежден и не может быть выдан без обслуживания;
- `UNUSABLE` — непригоден к эксплуатации.

### RentalAvailability

- `AVAILABLE` — доступен для выдачи;
- `RENTED` — выдан клиенту;
- `MAINTENANCE` — недоступен на время обслуживания или ремонта.

`RentalAvailability` описывает только операционную доступность экземпляра. Бронирование не
моделируется значением этого перечисления.

## Инварианты

1. Один `RentalAsset` соответствует одной физической вещи и никогда не содержит `quantity`.
2. Идентификатор и присвоенный номер экземпляра уникальны и не переиспользуются.
3. Выдать можно только экземпляр с:
   - `purpose == RENTAL`;
   - `availability == AVAILABLE`;
   - `condition` в множестве `NEW`, `GOOD`, `FAIR`.
4. `DAMAGED` и `UNUSABLE` блокируют выдачу.
5. Экземпляр с `purpose == SALE` или `RETIRED` нельзя выдать в аренду.
6. Выданный экземпляр нельзя вывести в продажу, отправить в обслуживание или списать до
   завершения возврата.
7. `RETIRED` является терминальным состоянием.
8. Вывод в продажу не удаляет `RentalAsset`, его идентификаторы или историю.
9. Текущие поля изменяются только через доменные команды.
10. История бизнес-операций не удаляется и не переписывается. Ошибка исправляется новой
    компенсирующей или административной операцией с аудитом.

## Доменные команды

### create

Создает один физический экземпляр при завершении Intake.

Допустимое исходное состояние: экземпляр еще не существует.

Начальное состояние:

```text
purpose = RENTAL
condition = NEW | GOOD | FAIR
availability = AVAILABLE
```

Если во время приемки зафиксировано `DAMAGED` или `UNUSABLE`, экземпляр создается с
`availability = MAINTENANCE` и не может быть выдан.

### checkout

Выдает экземпляр в аренду.

Допустимое исходное состояние:

```text
purpose = RENTAL
availability = AVAILABLE
condition = NEW | GOOD | FAIR
```

Результат:

```text
availability = RENTED
```

### accept_return

Принимает ранее выданный экземпляр и фиксирует его состояние после осмотра.

Допустимое исходное состояние:

```text
purpose = RENTAL
availability = RENTED
```

Результат:

- исправный экземпляр: `availability = AVAILABLE`, состояние `NEW`, `GOOD` или `FAIR`;
- поврежденный экземпляр: `availability = MAINTENANCE`, состояние `DAMAGED` или `UNUSABLE`.

Возврат не должен автоматически скрывать обнаруженное повреждение или возвращать поврежденный
экземпляр в доступный фонд.

### send_to_maintenance

Передает доступный экземпляр в обслуживание.

Допустимое исходное состояние:

```text
purpose = RENTAL
availability = AVAILABLE
```

Результат:

```text
availability = MAINTENANCE
```

Команда недопустима для выданного, выставленного на продажу или списанного экземпляра.

### complete_maintenance

Завершает обслуживание и фиксирует новое физическое состояние.

Допустимое исходное состояние:

```text
purpose = RENTAL
availability = MAINTENANCE
```

Результат:

- при `NEW`, `GOOD` или `FAIR`: `availability = AVAILABLE`;
- при `DAMAGED` или `UNUSABLE`: экземпляр остается недоступным, пока не будет выполнена следующая
  допустимая бизнес-операция.

### withdraw_for_sale

Выводит экземпляр из прокатного фонда для будущей продажи.

Допустимое исходное состояние:

```text
purpose = RENTAL
availability = AVAILABLE
condition != UNUSABLE
```

Результат:

```text
purpose = SALE
```

Команда не означает фактическую продажу. `RentalAsset` и вся его история сохраняются.

### retire

Окончательно выводит экземпляр из оборота с обязательной причиной.

Допустимое исходное состояние:

- экземпляр не выдан;
- экземпляр не участвует в активной операции;
- `purpose` не равен `RETIRED`.

Результат:

```text
purpose = RETIRED
```

Состояние является терминальным.

## История

Текущие `purpose`, `condition` и `availability` изменяемы, но только через доменные команды.

История выдач, возвратов, обслуживания, вывода в продажу и списания является
append-only: существующие факты не удаляются и не переписываются.

## Отложенные расширения

В Sprint 9 не входят:

- бронирование и сущность `Reservation`;
- договоры аренды;
- залоги, платежи, штрафы и просрочки;
- заказы на обслуживание;
- фактическая продажа экземпляра и интеграция с будущим доменом Sales.
