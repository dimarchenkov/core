# Core

## Назначение

Core — операционная платформа для небольшого магазина и сервиса аренды: управление номенклатурой, физическими объектами, складом, фото, ценами, работой сотрудников и публикацией во внешние каналы.

Core является единым источником достоверных данных для 2010shop.

В долгосрочной перспективе Core должен поддерживать самостоятельное развёртывание, доступ по подписке и поставку в составе франшизы. Бизнес-возможности доступны через API, поэтому пользователь или партнёр может создать собственный интерфейс.

Внешние системы — AQSI, Tilda, Telegram, Мессенджер Макс, Wildberries, Яндекс Маркет и другие — не являются главным местом хранения данных. Они получают данные из Core.

## Главная цель

Сделать быстрый и надежный процесс:

1. Товар или арендный объект поступил.
2. Его сразу сфотографировали: первичное фото начинает процесс.
3. Создали или нашли карточку.
4. Присвоили SKU и штрихкод.
5. Указали количество, цену, источник поступления.
6. Отправили товар в кассу.
7. Подготовили карточку для Tilda.
8. При необходимости опубликовали пост в Telegram.
9. Для арендного экземпляра создали RentalAsset и продолжили отдельный жизненный цикл выдачи, возврата, осмотра и обслуживания.

## Ключевые принципы

### Core — источник истины

Все данные о товаре создаются и изменяются в Core.

### Photo First

Новый физический объект появляется в рабочем процессе с первой фотографии. Товар без фото не считается полноценно принятым.

Карточку нельзя перевести в статус готовности к публикации или отправке в кассу, если у товара или варианта нет хотя бы одного фото.

### Product и Variant разделены

Product — это общая карточка товара.

Variant — конкретная складская позиция: цвет, размер, комплектация, SKU, штрихкод, остаток и цена.

Остатки, SKU и штрихкоды живут на уровне Variant.

### RentalAsset для аренды

Для аренды используется отдельная сущность RentalAsset — физический экземпляр оборудования.

Например, один Product “Шуруповерт Bosch” через свои Variant может иметь несколько RentalAsset:

- REN-000001
- REN-000002
- REN-000003

Каждый RentalAsset имеет собственный статус, историю, состояние, фото осмотра и пломбы.

Аренда является первоклассным жизненным циклом Core наряду с продажей, а не необязательной внешней надстройкой.

### Действия имеют автора и измеряются

Важные действия сохраняют автора и время. Эти данные используются не только для расследований, но и для операционной аналитики, обучения и контроля работы сотрудников.

### API First

Полноценная бизнес-возможность должна быть доступна через документированный API. SQLAdmin, мобильный интерфейс, будущий frontend и партнёрские интерфейсы являются клиентами Core.

### Простое развёртывание и документация

Развёртывание и bootstrap должны быть воспроизводимыми, а документация — частью продукта для пользователя, администратора, разработчика, франшизы и инвестора.

### Внешние системы вторичны

AQSI, Tilda, Telegram, WB и Яндекс Маркет являются каналами публикации или продаж.

Логика Core не должна зависеть от конкретной внешней системы.

### Модульность

Каждая интеграция должна быть отдельным модулем.

Отключение Tilda, AQSI или любого маркетплейса не должно ломать учет товаров.

### История изменений

Важные действия должны сохраняться:

- создание товара;
- изменение цены;
- изменение остатка;
- приемка;
- списание;
- публикация;
- выдача в аренду;
- возврат из аренды;
- изменение состояния оборудования.

### Mobile-first приемка

Core должен позволять быстро принять товар с телефона:

- открыть страницу приемки;
- сделать фото;
- заполнить основные данные;
- сохранить черновик.

Первый интерфейс приемки должен быть удобен с телефона.

Полный набор продуктовых принципов зафиксирован в [docs/01_principles.md](01_principles.md).

## Не входит в первую версию

Core на старте не должен заменять бухгалтерию, CRM, полноценную ERP или маркетплейс-кабинеты.

Первая цель — надежный учет товаров, фото, остатков, цен, штрихкодов и подготовка к публикации.
