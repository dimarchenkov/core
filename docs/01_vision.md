# Core

## Назначение

Core — внутренняя система управления номенклатурой, складом, фото, ценами, арендой и публикацией товаров во внешние каналы.

Core является единым источником достоверных данных для 2010shop.

Внешние системы — AQSI, Tilda, Telegram, Мессенджер Макс, Wildberries, Яндекс Маркет и другие — не являются главным местом хранения данных. Они получают данные из Core.

## Главная цель

Сделать быстрый и надежный процесс:

1. Товар поступил.
2. Его сфотографировали.
3. Создали карточку.
4. Присвоили SKU и штрихкод.
5. Указали количество, цену, источник поступления.
6. Отправили товар в кассу.
7. Подготовили карточку для Tilda.
8. При необходимости опубликовали пост в Telegram.

## Ключевые принципы

### Core — источник истины

Все данные о товаре создаются и изменяются в Core.

### Фото обязательно

Товар без фото не считается полноценно принятым.

Карточку нельзя перевести в статус готовности к публикации или отправке в кассу, если у товара или варианта нет хотя бы одного фото.

### Product и Variant разделены

Product — это общая карточка товара.

Variant — конкретная складская позиция: цвет, размер, комплектация, SKU, штрихкод, остаток и цена.

Остатки, SKU и штрихкоды живут на уровне Variant.

### Asset для аренды

Для аренды используется отдельная сущность Asset — физический экземпляр оборудования.

Например, один Product “Шуруповерт Bosch” может иметь несколько Asset:

- REN-000001
- REN-000002
- REN-000003

Каждый Asset имеет собственный статус, историю, состояние, фото осмотра и пломбы.

### Внешние системы вторичны

AQSI, Tilda, Telegram, WB и Яндекс Маркет являются каналами публикации или продаж.

Логика Core не должна зависеть от конкретной внешней системы.

### Модульность

Каждая интеграция должна быть отдельным модулем.

Отключение Tilda, AQSI или любого маркетплейса не должно ломать учет товаров.

### История изменений

Важные действия должны сохраняться:

- создание товара;
- изменение цены;
- изменение остатка;
- приемка;
- списание;
- публикация;
- выдача в аренду;
- возврат из аренды;
- изменение состояния оборудования.

### Mobile-first приемка

Core должен позволять быстро принять товар с телефона:

- открыть страницу приемки;
- сделать фото;
- заполнить основные данные;
- сохранить черновик.

Первый интерфейс приемки должен быть удобен с телефона.

## Не входит в первую версию

Core на старте не должен заменять бухгалтерию, CRM, полноценную ERP или маркетплейс-кабинеты.

Первая цель — надежный учет товаров, фото, остатков, цен, штрихкодов и подготовка к публикации.
