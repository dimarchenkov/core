# Core MVP

## Цель MVP

Первая версия Core должна решить одну задачу:

> Позволить быстро принять товар, сохранить его в собственной базе данных и подготовить к продаже.

Если для этого не нужна какая-либо функция — она не входит в MVP.

---

# Основной сценарий

```text
Поступил товар

↓

Сделали фото

↓

Создали карточку

↓

Получили SKU

↓

Получили штрихкод

↓

Добавили количество

↓

Добавили цены

↓

Сохранили

↓

Отправили в AQSI
```

Если этот сценарий работает быстро и надежно — MVP считается успешным.

Минимальная авторизация.

Роли:

- Administrator
- Manager

- Импорт CSV из Tilda

---

## Интеграции

### AQSI

Первая полноценная интеграция.

Поддерживается:

- создание товара
- обновление товара

---

### Tilda

На MVP отсутствует.

Карточка готовится в Core.

Публикация выполняется вручную.

Автоматизация будет реализована позже.

---

# Что НЕ входит в MVP

## Rental

Не входит в текущий продающий вертикальный срез, но является первоклассным направлением продукта. Архитектурные решения Ready for Sale и Sell не должны блокировать будущий жизненный цикл RentalAsset: выдача, возврат, осмотр и обслуживание.

---

## React

SQLAdmin полностью покрывает административную часть MVP.

React разрабатывается после стабилизации backend.

---

## Wildberries

Не входит.

---

## Яндекс Маркет

Не входит.

---

## MAX

Не входит.

---

## Telegram

Не входит.

---

## Несколько складов

Пока один склад.

Архитектура должна поддерживать несколько складов без изменения моделей.

Будут реализованы позже.

# Требования к производительности

Создание нового товара должно занимать менее одной минуты.

Создание товара с телефона должно занимать менее двух минут.

---

# Definition of Done

MVP считается готовым, если можно:

✅ открыть Core

✅ авторизоваться

✅ создать категорию

✅ создать товар

✅ добавить вариант

✅ сфотографировать товар

✅ автоматически получить SKU

✅ автоматически получить штрихкод

✅ добавить закупочную цену

✅ добавить розничную цену

✅ принять товар на склад

✅ увидеть остаток

✅ отправить товар в AQSI

---

# Главный принцип MVP

Работающий процесс важнее количества функций.

Лучше одна полностью рабочая цепочка:

Приемка → Каталог → AQSI

чем десять недоделанных модулей.

---

# После MVP

После готовности основного пути продажи Core развивает второй равноправный жизненный цикл — аренду — и внешние каналы.

Приоритеты:

1. Rental lifecycle
2. Employee activity and operational analytics
3. Tilda Sync
4. Telegram
5. MAX
6. Optional first-party frontend
7. Wildberries
8. Яндекс Маркет
