# Core Product Philosophy

## Что такое Core

Core — операционная платформа для небольших магазинов и сервисов аренды.

Она проводит физический объект через весь рабочий цикл: от первой фотографии и регистрации до продажи, повторной аренды, обслуживания, публикации во внешние каналы и анализа работы команды.

Core не является бухгалтерией, универсальной ERP или обязательным пользовательским интерфейсом. Он предоставляет надёжные бизнес-процессы и API, поверх которых могут работать SQLAdmin, мобильный интерфейс, собственный frontend, мессенджеры и внешние интеграции.

## Продуктовая цель

Core должен позволить владельцу перестать быть незаменимым участником ежедневных операций.

Новый сотрудник должен быстро освоить понятные рабочие режимы, а владелец — видеть:

- что было сделано;
- кто это сделал;
- сколько времени занял процесс;
- где произошли ошибки или отмены;
- что требует внимания.

## Два равноправных жизненных цикла

### Продажа

```text
Фото → Каталог → Приход → Подготовка к продаже → Продажа
```

### Аренда

```text
Фото → Каталог → Asset → Выдача → Возврат → Осмотр → Обслуживание → Следующая выдача
```

Продажа и аренда используют общую основу: каталог, варианты, изображения, идентификаторы, местоположение, пользователей, аудит и историю операций. Аренда является первоклассным жизненным циклом Core, а не внешним дополнением.

## Модель поставки продукта

Core должен поддерживать несколько способов использования:

- самостоятельное развёртывание;
- управляемый доступ по подписке;
- поставка в составе франшизы;
- подключение собственного интерфейса через API.

Новые возможности не должны жёстко связывать бизнес-логику с конкретным интерфейсом или способом поставки.
